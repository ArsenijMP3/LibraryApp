# LibraryApp — Модуль 2 (Задания 2.2, 2.3, 2.4)

Десктоп-приложение на C# / WinForms (.NET 6), подключение к Access (`base.accdb`) через OLE DB.

## Что реализовано

- **2.2 Оформление и стиль** — единый стиль окон (тёмная верхняя панель, акцентный цвет #2E8B57),
  заголовки окон соответствуют назначению, нейминг PascalCase/camelCase для C#, одна команда в строке.
- **2.3 Авторизация** — окно входа открывается первым (`LoginForm`), вход по логину/паролю из `Читатели`,
  кнопка «Войти как гость», после входа — главный экран (`MainForm`) с ФИО и ролью в правом верхнем углу,
  кнопка «Выход» возвращает на окно входа.
- **2.4 Список записей** — каталог книг из БД с полями: название, автор, жанр, год издания, издательство,
  кол-во экземпляров, описание, наличие. Раскраска строк:
  - 0 экземпляров → красный;
  - в наличии → зелёный (#2E8B57);
  - выдана (есть активная выдача) → голубой.

> Поиск/сортировка/фильтрация (3.2) и формы добавления/редактирования/удаления (3.3, 3.4) — это Модуль 3,
> в этой версии не реализованы намеренно.

## Как запустить

1. Установите **.NET 6 SDK** (если ещё нет): https://dotnet.microsoft.com/download/dotnet/6.0
2. Если на машине не установлен Access/Office (а значит и Access Database Engine) —
   поставьте **Microsoft Access Database Engine 2016 Redistributable** (нужна 64-разрядная версия,
   если ваш .NET SDK 64-битный — обычно так и есть).
3. Скопируйте ваш файл `base.accdb` в корень проекта (рядом с `LibraryApp.csproj`) —
   он автоматически скопируется в папку сборки при каждой компиляции.
4. В VS Code, в терминале, в папке проекта:
   ```bash
   dotnet restore
   dotnet build
   dotnet run
   ```
5. Для проверки входа используйте логины из `Скрипт_заполнения.sql`, например:
   `sidorova` / `pass456` (читатель), `kuznecov` / `lib789` (библиотекарь),
   `morozova` / `admin123` (администратор).

## Важное замечание по тестовым данным

В `Скрипт_заполнения.sql` читатель «Петров Сергей Иванович» имеет `роль_id = 1` («гость»),
хотя у гостя по сценарию не должно быть логина/пароля. Рекомендую поменять ему роль на
«читатель» (`роль_id = 2`), иначе при тестировании авторизации это будет сбивать с толку.
